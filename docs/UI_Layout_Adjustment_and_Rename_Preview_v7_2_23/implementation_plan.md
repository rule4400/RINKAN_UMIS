# UMIS Pro v7.2.23 UIレイアウト調整とリネームプレビュー実装計画

## 概要
ユーザーの利便性向上のため、保存先選択ボタンの配置を変更し、取り込み後のファイル名が事前に確認できる「リネームプレビュー」機能を実装します。

## 変更内容

### 1. 保存先選択ボタンの移動
- **現状**: [アイコン] [保存先:] [パス入力欄] [フォルダ選択ボタン]
- **変更後**: [アイコン] [保存先:] [フォルダ選択ボタン] [パス入力欄]
- `build_ui` 内の `dest_row` のコントロール順序を入れ替えます。

### 2. リネームプレビューの実装
- **新規ラベル**: `self.lbl_rename_preview` を作成し、プログレスバーの上部（`lbl_status` の付近）に配置します。
- **プレビューロジック**:
    - `update_preview` メソッド内で、現在選択されている、または割り当てられているファイルの情報を元に、リネーム後のファイル名を1件シミュレーション表示します。
    - シミュレーションには既存の `self.cfg_mgr.generate_filename` を利用します。
    - 日付は現在時刻、連番は `0001`、ファイル名は `sample.jpg` として表示します。

## 検証計画
- アプリケーションが起動することを確認。
- 保存先の「フォルダを開く」アイコンがパス入力欄の左側に移動していることを確認。
- 会場、カメラマン、シーンが選択された際、プログレスバーの上に「リネーム例: 260324_会場_101_シーン_カメラマン_sample.jpg」のようなプレビューが表示されることを確認。
- 設定（リネーム規則）を変更した際に、プレビュー内容が即座に更新されることを確認。
