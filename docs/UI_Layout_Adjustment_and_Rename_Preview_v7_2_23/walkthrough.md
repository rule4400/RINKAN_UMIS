# 修正内容の確認 (v7.2.23)

## 実施内容
保存先選択の操作性向上と、取り込み後のファイル名を事前に確認できるプレビュー機能を実装しました。

### 1. 保存先選択ボタンのレイアウト調整
- `build_ui` 内の保存先入力行（`dest_row`）において、フォルダ選択ボタン（フォルダを開くアイコン）をパス入力欄の左側に移動しました。
- これにより、ユーザーが「まずボタンを押してフォルダを選ぶ」という自然な視線誘導・操作フローに適合させました。

### 2. リネームプレビュー機能の実装
- プログレスバーの上部に `lbl_rename_preview` を追加しました。
- 会場、カメラマン、シーンが選択されると、現在のリネーム設定（日付書式、会場名・シーン名・連番の有無など）に基づいた「リネーム後のファイル名サンプル」をリアルタイムで表示します。
- 設定（リネーム規則）を変更した際にも即座にプレビューが更新されるため、設定ミスを未然に防ぐことができます。

## 検証結果
- アプリケーションが正常に起動することを確認。
- 保存先のボタンがパスの左側に配置されていることを確認。
- 条件（ドライブ、カメラマン、シーン割当）が揃うと、具体的なファイル名の例（例: `260424_会場_101_シーン名_カメラマン_sample.jpg`）が表示されることを確認。
- 設定画面でリネーム項目をOFFにすると、プレビューからも該当項目が消えることを確認。
