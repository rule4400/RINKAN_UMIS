# UMIS Pro v7.2.26 撮影者・カードID選択UI刷新および設定モーダルUX改善計画

## 概要
識別情報の選択フローを左右2カラム構成の直感的なUIに刷新し、設定モーダル内での操作が途切れないシームレスなUXを提供します。また、カードID管理を完全に独立化させます。

## 変更内容

### 1. 識別情報選択UIの刷新 (`show_identity_modal`)
- **レイアウト**: `ft.Row` を使用し、左側に「カメラマン選択」、右側に「カードID選択」を配置します。
- **選択挙動**: 
    - 開いた直後はどちらも未選択状態。
    - 項目クリックでハイライト（背景色変更）。
    - 両方を選択した瞬間にオートクローズし、メイン画面を更新。
- **管理機能**: 各カラムの下部に「＋」ボタンを配置し、その場で追加・削除を可能にします。

### 2. 設定モーダルのUX改善
- **ヒント表示**: `ft.Tooltip` を導入し、設定項目にカーソルを合わせるか長押しすることで説明が表示されるようにします。これにより、別ダイアログを開く手間を省きます。
- **リスト編集の非クローズ化**: 
    - `_open_list_editor` を呼び出す際、親の設定モーダルを閉じないように変更します。
    - リスト編集後の変更が親モーダルに（必要であれば）反映されるよう、更新ロジックを調整します。

### 3. データ管理の統合
- カードIDを `ConfigManager` で管理されるリスト方式に一本化します。
- リネーム規則において、選択されたカードIDが正しく反映されることを保証します。

### 4. 既存機能の維持
- 会場選択の保存、右クリック範囲選択、データ保存ルール（指定フォルダ直下）などの重要機能を維持します。

## 検証計画
- アプリケーションが正常に起動することを確認。
- 識別情報選択ボタンから2カラムのモーダルが開き、両方選択で自動的に閉じることを確認。
- 設定モーダル内の「i」アイコンがツールチップとして機能することを確認。
- 設定モーダルからリスト編集（カメラマン等）を開いても、元の設定モーダルが維持されることを確認。
- 最終的なリネーム結果が期待通りであることを確認。
