# 実装計画 - v10.2.0 UI修正

## 目標

スクリーンショットのようなメディア管理UIを実現する。

### 目指すUI要件
1. **サムネイル**: 動画の1フレーム目を切り出して表示（キャッシュなければ生成）
2. **再生時間バッジ**: 全動画ファイルの右下に `0:00` 形式でバッジ表示
3. **選択状態の視覚表示**: クリックしたタイルに青い枠を表示（フォーカスとして）
4. **インスペクター**: ファイル選択時に右側パネルにプレビューとファイル情報を表示

---

## 特定した根本問題

### 問題1: サムネイルが表示されない / バッジが付かない
- **原因A**: `_generate_thumbnail` で動画のサムネイル生成に失敗していても例外が握り潰されている
- **原因B**: キャッシュヒット時にdurationの取得が不確実（DBにもなければ取得しない）
- **原因C**: `_failed_files` に登録されると以後永続スキップ → 修正済み(v10.1.0)
- **原因D**: `_thumb_worker` でエラー時に `task_done()` が呼ばれずキューが詰まる → 修正済み(v10.1.0)

### 問題2: 選択状態（青い枠）が表示されない
- **原因**: `on_file_click` でトグル式の `selected` フラグ（true/false）を管理しているが、
  フォーカス（どれを今見ているか）と混在しており、クリックするたびに枠が消える

### 問題3: インスペクターが更新されない
- **原因**: インジェストモードでは `update_col_preview` が呼ばれない分岐になっていた（v10.1.0で部分修正済み）

---

## 修正方針

### 修正1: タイルのクリック挙動を「フォーカス」に統一
- `on_file_click` で `selected` のトグルをやめ、`focused_file_index` への代入のみにする
- フォーカスされたタイルは常に青い枠が表示される
- `_get_item_style` でフォーカス時の枠色・太さを最優先にする

### 修正2: インスペクター更新を常時実行
- `on_file_click` 内で `update_col_preview(switch_to_preview=True)` を常に呼び出す
- segfault 対策として、動画プレイヤーの生成はデバウンスを経由させる

### 修正3: サムネイル生成ロジックの強化
- `qlmanage` が失敗した場合も `ffmpeg` で確実にフォールバック
- `ffmpeg` のタイムアウトを適切に設定（5秒）
- 生成失敗時のログを出力して問題を可視化

### 修正4: バッジ更新の確実化
- `_thumb_worker` で `_update_item_visual(idx)` を必ず呼び出す（条件なし）
- `_update_item_visual` でコンテナ更新を確実に実行

## 検証計画

1. アプリ起動後、ライブラリのフォルダをクリック
2. 動画ファイルのサムネイルが逐次表示されること
3. 全ての動画ファイルに再生時間バッジが付くこと
4. タイルをクリックすると青い枠が表示されること（他のタイルの枠は消える）
5. インスペクターに動画プレビューとファイル情報が表示されること
