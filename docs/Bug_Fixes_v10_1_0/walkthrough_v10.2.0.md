# 修正内容の確認 (Walkthrough) - v10.2.0

## 概要
ユーザーが提示したスクリーンショット（動画管理UIイメージ）に合わせ、以下3点を実現するためにv10.2.0を作成しました。

- サムネイルに動画1フレームを表示 + 右下に再生時間バッジ  
- タイルクリック時に青い枠（フォーカス）を常時表示  
- 右側インスペクターにプレビュー + ファイル情報を表示  

---

## 修正内容

### 1. `on_file_click` の挙動変更（最重要）

**問題**: クリックのたびに `selected` がトグルされていたため、青い枠が点滅して消えていた。

**修正**: 
- インジェストモードでは `selected` トグルを維持（シーン割当に使うため）
- **セレクトモードでは `focused_file_index` の更新のみ**に統一
- `_update_item_visual` で `focused_file_index == idx` を見て枠を描画するので、クリックした時点から枠が固定される

### 2. `_get_item_style` のフォーカス優先度変更

**問題**: `is_focused` よりも他の状態（`is_flagged`, `is_sel`）が先に評価されていた。

**修正**: 両モードともフォーカスを最優先で評価するよう変更。青枠 (`COLOR_PRIMARY`) が最前面に表示される。

### 3. `update_col_preview` のモード別対応

**問題**: インジェストモードではインスペクタ更新関数が呼ばれなかった。

**修正**: 全モード共通でデバウンス (50ms) 経由でインスペクタを更新。
- セレクトモード → `_update_inspector_details()` を呼び出す
- インジェストモード → `_build_preview_widget()` を呼び出して `col_preview_area` を更新

### 4. v10.1.0 からの継続修正（バッジ・サムネイル）

v10.1.0 で実施した以下の修正は v10.2.0 にも引き継がれています。
- キャッシュヒット時でも ffprobe でdurationを必ず取得
- `_thumb_worker` の `task_done()` を `finally` で必ず呼び出す
- `_failed_files` をリセット時にクリアして永続スキップを防止
