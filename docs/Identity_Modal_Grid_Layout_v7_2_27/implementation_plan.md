# UMIS Pro v7.2.27 識別情報選択UIのモダン化・レイアウト最適化計画

## 概要
識別情報選択モーダル（カメラマン・カードID）のデザインをタイル型レイアウトに刷新し、あわせて発生している不具合（Tooltipのエラーおよび初期化処理のNameError）を修正します。

## 修正および実装内容

### 1. バグ修正
- **Tooltipエラー (`TypeError: Tooltip.__init__() got an unexpected keyword argument 'content'`)**:
  - `create_info_btn` 関数を修正し、`ft.Tooltip` を直接使うのではなく、`ft.Icon` の `tooltip` プロパティを利用する形式に変更します。
- **初期化確認時のNameError (`NameError: cannot access free variable 'show_confirm_1_or_ssd_warning'`)**:
  - `format_card` メソッド内で、未取り込みファイルの警告時に `show_confirm_1_or_ssd_warning` を呼び出していますが、定義が下にあるためエラーになっています。この関数の定義順序を見直し、呼び出し前に定義されるようにします。

### 2. 識別情報選択UIのモダン化（タイル型レイアウト）
- **グリッド表示化**:
  - `show_identity_modal` 内の `pg_list_area` および `cid_list_area` を `ft.Column` から `ft.Row(wrap=True)`（折り返しありのRow）に変更し、横幅を有効活用するタイル配置にします。
- **タイルデザインの適正化**:
  - 各項目（Container）を固定幅（約110px）、固定高さ（約40px）に設定し、テキストを中央揃えにします。
  - 装飾として `border_radius=8`、`border=ft.border.all(1, COLOR_DIVIDER)`、`bgcolor="#2C2C2E"` を適用します。
  - 選択時は枠線を白色（太さ2）にし、背景色をプライマリ/アクセントカラーで強調します。
  - `ink=True` でクリック時の波紋エフェクトを有効化します。
- **追加ボタンの配置変更**:
  - 各カラムのリスト下部にあった「追加/編集」ボタンを、ヘッダー（「カメラマン」等の横）に小さなアイコンボタン (`ft.IconButton(ft.Icons.ADD_CIRCLE_OUTLINE)`) として移動します。
- **自動クローズの維持**:
  - 両方のカラムで項目が1つずつ選択された瞬間にモーダルを閉じるロジックを正確に維持します。

## 既存機能の維持
- 右クリック範囲選択、撮影データフォルダ非作成、初期化安全制限などのロジックは変更せず維持します。
- 「アクションしても閉じない設定モーダル」の挙動を担保します。

## 検証計画
- `v7.2.27` を起動し、先ほど発生したエラーが解消されていることを確認します。
- 設定画面を開き、ツールチップが正しく表示されることを確認します。
- 「カメラマン/カードID」ボタンをクリックし、新しいタイル型レイアウトが表示され、正しく選択および自動クローズが行われるかテストします。
- SDカード（またはダミー）の初期化プロセスを実行し、NameErrorが発生せずに確認ダイアログが表示されるかテストします。
