# RINKAN UMIS (Universal Media Ingest System) システム仕様書

## 1. 概要
RINKAN UMIS は、イベント撮影現場でのメディア取り込み（インジェスト）および選別作業を最適化するために設計されたアプリケーションです。Fletフレームワークを使用したモダンなGUIを提供し、大量の画像・動画ファイルを迅速に整理・リネーム・バックアップすることが可能です。

---

## 2. システム構成

### 2.1 技術スタック
- **言語**: Python 3.x
- **GUIフレームワーク**: Flet (Flutterベース)
- **画像処理**: Pillow (PIL)
- **システム操作**: psutil, subprocess (ffprobe等)
- **データ形式**: JSON (設定、履歴管理)

### 2.2 外部依存ツール
- **ffprobe**: 動画ファイルのメタデータ（解像度、フレームレート、コーデック、再生時間）取得に使用。
- **shutil / os**: ファイルのコピー、移動、ディレクトリ操作に使用。

---

## 3. 主要クラスと役割

### 3.1 `RinkanUMISApp`
アプリケーションのメインクラス。UIの構築、イベントハンドリング、各ワーカーの制御を統合管理します。

### 3.2 `ConfigManager`
プロジェクトごとの設定（リネーム規則、シーンリスト、除外フォルダ、パス設定など）を管理します。設定は `projects/` フォルダ内のJSONファイルとして保存されます。

### 3.3 `CopyWorker` (threading.Thread)
「取り込み（Ingest）」処理を担当。
1. フェーズ1: ソースメディアから一時作業フォルダ（TEMP_DIR）へファイルを高速転送。
2. フェーズ2: 設定された規則に基づきリネームを行い、保存先フォルダへ整理・検証。

### 3.4 `SelectWorker` (threading.Thread)
「選別（Select）」モードでのファイル出力を担当。指定されたシーンフォルダ内のサブフォルダ（例：「選別」）へファイルをコピーします。

### 3.5 `HistoryLogger`
月単位で実行履歴（取り込み日時、ファイル数、サイズ、ステータス等）を記録します。

### 3.6 `FormatWorker`
OS標準のコマンドを使用して、メディア（SDカード等）のクイックフォーマット（ExFAT）を実行します。

## 4. 機能仕様

### 4.1 取り込みモード (Ingest Mode)
- **メディア検出**: 接続されたドライブを自動スキャンし、画像・動画・RAW・音声を自動分類。
- **シーン割当**: ドラッグ選択または範囲選択したファイルを、定義済みのシーン（例：101_開校式）にワンクリックで割り当て。
- **リネーム規則**: 以下の要素を組み合わせ可能。
  - 撮影日 (date) / 会場名 (location) / シーン名 (scene) / カメラマン名 (photographer) / カードID (card_id)
- **重複防止**: 保存先に同名ファイルが存在する場合、スキップまたは連番付与で保護。

### 4.2 セレクトモード (Select Mode)
- **ライブラリ表示**: 取り込み済みの保存先ディレクトリをスキャンし、シーンごとに閲覧可能。
- **クイックプレビュー**: スペースキーで全画面表示。動画・音声の再生に対応。
- **選別フラグ**: 特定のファイルにフラグを立て、一括で選別フォルダへ出力。

### 4.3 キャッシュシステム
- **thumbnails/**: 一覧表示用の軽量サムネイルを生成・保存。
- **previews/**: クイックプレビュー用の高解像度（1080p相当）画像を生成。
- **duration_cache.json**: 動画の再生時間を記録し、再スキャン時のffprobe呼び出しを最小化。

## 5. データ管理構造

### 5.1 設定ファイル (JSON)
`projects/[プロジェクト名].json`
- `scenes`: シーンリスト（Day, No, Name）
- `options`: リネームやフォルダ作成のスイッチ状態
- `paths`: 保存先および一時フォルダのパス

### 5.2 実行履歴 (JSON)
`history/history_YYYYMM.json`
- 実行ごとの統計情報（成功/失敗/サイズ）を保持。

---

## 6. ディレクトリ構造（標準設定）
- **[保存先ルート] / [シーン名] / [カテゴリ] / [カメラマン] / [リネーム済みファイル]**
  - カテゴリ例: 動画 (Movie), 写真 (Photo), Raw, 音声 (Audio)
  - 選別サブフォルダ例: [シーン名] / 選別 /
