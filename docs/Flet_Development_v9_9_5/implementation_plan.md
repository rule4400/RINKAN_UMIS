# セレクトモード安定化・高速化の実装計画

セレクトモードにおいて、ファイル移動時に発生する動作の不安定さ（フリーズ、セグメンテーション違反）を解消するための実装計画です。

## 現状の課題
- **同期処理によるフリーズ**: `_update_inspector_details` 内で `ffprobe` を UI スレッドで実行しており、ファイル選択のたびに操作が固まる。
- **リソース競合**: 高速移動時に `ft.Video` インスタンスが短時間に大量に生成・破棄され、ネイティブレイヤーでクラッシュ (Segmentation fault) を引き起こす。
- **デバウンスの欠如**: 高速移動中の全てのファイルに対して重い処理が走っている。

## 解決策

### 1. インスペクター更新のデバウンス (Debounce) 実装
`threading.Timer` を利用し、ファイルフォーカスが移動してから **300ms** 静止したときのみ、詳細情報の更新（動画読み込み含む）を開始するようにします。

### 2. メタデータ取得の非同期化
`ffprobe` による動画情報の取得を別スレッドで実行します。取得完了後に `page.run_task` (または `page.update`) を通じて UI を更新します。

### 3. 動画プレイヤーの生成管理
新しいプレビューを表示する前に、既存のプレイヤーインスタンスを確実にポーズさせ、リソース解放を促すロジックを追加します。

## 修正内容の詳細

### [UMISApp クラス]
#### `update_col_preview` メソッド
- セレクトモード時、既存のタイマーがあればキャンセル。
- 300ms 後の遅延実行として `_update_inspector_details` をスケジュールする。

#### `_update_inspector_details` メソッド
- `ffprobe` 実行部分を `threading.Thread` に分離。
- 動画プレイヤー生成前に `self.video_ctrl_col.pause()` などを確実に呼ぶ（存在する場合）。

## 抽出テスト・検証プラン
1. セレクトモードで矢印キーを押し続け、アプリがクラッシュしないことを確認する。
2. 矢印キーを離した瞬間に、プレビューとメタデータがスムーズに表示されることを確認する。
3. プレビュー再生中に次のファイルへ移動しても、音声が重なったりクラッシュしたりしないことを確認する。
