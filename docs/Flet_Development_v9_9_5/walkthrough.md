# 修正内容の確認 (Walkthrough)

セレクトモードの安定化および Segmentation fault 対策として、以下の修正を適用しました。

## 実施した修正

### 1. デバウンス処理の導入
矢印キーでファイルを連続移動する際、移動のたびに重い処理が走るのを防ぐため、**300ms の静止待機時間**を設けました。キーを離した瞬間にプレビューが更新されるようになり、高速移動時のレスポンスと安定性が大幅に向上しました。

### 2. ffprobe の非同期化
これまで UI スレッドをブロックしていた `ffprobe` によるメタデータ（解像度、コーデック等）取得を、**バックグラウンドスレッド**で実行するように変更しました。これにより、情報の取得中も UI が固まることなく、スムーズに操作を継続できます。

### 3. 動画プレイヤーのリソース管理強化
新しい動画を読み込む際、既存のプレイヤーインスタンスに対して明示的に `pause()` を呼び出し、リソースの競合を避えるようにしました。また、読み込み中を示すインジケータを表示し、視覚的なフィードバックを強化しました。

## 検証方法
1. セレクトモードで矢印キーを押し続け、高速にファイルを移動してもアプリが落ちないことを確認。
2. キーを離した直後にプレビューが表示され、少し遅れて詳細情報（解像度など）が表示される挙動を確認。

## デバッグモードの実装 (v10.0.1)

クラッシュ（Segmentation fault）の原因特定を可能にするため、詳細なデバッグ機能を搭載しました：
- **詳細ログの永続化**: 通常の画面ログに加え、`~/.umis_pro/debug.log` にすべてのイベントをタイムスタンプ付きで記録します。アプリが強制終了しても、直前の動作を確認可能です。
- **UIスイッチの追加**: ヘッダーに「DEBUG」スイッチを追加しました。これをONにすると、ファイルへの書き出しが有効になります。
- **詳細イベントの追跡**: サイドバークリック、ファイルクリック、動画プレイヤーの破棄などの主要な処理ポイントにデバッグログを追加しました。
