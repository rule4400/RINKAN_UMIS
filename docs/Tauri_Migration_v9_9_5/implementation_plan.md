# UMIS Tauri 移行実施計画書 (v9.9.5ベース)

Flet（Python）からTauri（Rust + Next.js）へ移行し、大量 of ファイル処理と動画再生のパフォーマンスを劇的に改善します。

## ユーザーレビューが必要な事項

> [!IMPORTANT]
> **アーキテクチャの変更**
> ロジックをPythonからRustに移植します。これにより速度は向上しますが、既存のPythonライブラリの一部（特定の画像解析など）をRust版に置き換える必要があります。

> [!TIP]
> **UIフレームワークの選定**
> プレミアムな外観を実現するため、Next.js + Tailwind CSS + Shadcn UI を提案します。Lucide Iconsを使用し、現在のFlet版よりも洗練されたデザインを目指します。

## オープンクエスチョン

- **既存データの移行**: `metadata_v1.db` (SQLite) のデータ構造は維持しつつ、Rustからアクセスするようにしますが、過去の履歴データもすべて引き継ぐ必要がありますか？
- **動画再生の要件**: 4K動画や特定のコーデック（HEVCなど）の再生が必要な場合、OS側のデコーダーに依存しますが、特別な再生要件はありますか？

## 提案される変更

### [プロジェクト基盤]

#### [NEW] `umis-tauri/`
新規プロジェクトフォルダを作成し、Tauri環境を初期化します。

### [バックエンド (Rust)]

#### [NEW] `src-tauri/src/main.rs`
- **File Commands**: 大量ファイルの高速スキャン、一括コピー、リネームロジックをRustで実装。
- **Drive Monitoring**: OSネイティブなAPIを使用してドライブの接続・解除をリアルタイムに検知。
- **Thumbnail Engine**: Rustの `image` クレートを使用して、Pythonよりも高速にサムネイルを生成。

### [フロントエンド (Next.js / React)]

#### [NEW] `src/`
- **Layout**: グラスモーフィズムやスムーズなアニメーションを取り入れたモダンなUI。
- **Media Player**: HTML5 Video APIを使用した低遅延な動画プレビュー。
- **Virtual List**: 何万件ものファイルがあってもカクつかない仮想スクロールの実装。

## 変更の理由
- **パフォーマンス**: Flet/Pythonのブリッジによる遅延を排除し、Rustによる並列処理でファイル操作を高速化します。
- **動画再生**: Webviewの最適化されたメディアエンジンを使用することで、MP4再生のスタッタリングを解消します。

## 検証計画

### 自動テスト
- `cargo test`: Rust側のファイル操作ロジックの単体テスト。
- `npm run test`: Reactコンポーネントのテスト。

- 1,000枚以上の写真を含むドライブを接続し、スキャンとプレビューの速度を現行版（v9.9.5）と比較。
