# UMIS Pro v7.6.0 アップデート実装計画

## 概要
`rinkan_umis_pro_v7.5.2.py` をベースに、提供された新しいロジックを適用し、UIのレスポンシブ対応と機能改善を行った `rinkan_umis_pro_v7.6.0.py` を作成します。

## ユーザーレビューが必要な項目
なし（提供されたコードをそのまま適用するため）

## 変更内容

### [UMIS Pro 本体]

#### [NEW] [rinkan_umis_pro_v7.6.0.py](file:///Users/kiroku_keizo/開発/UMIS (Universal Media Ingest System)/安定版/rinkan_umis_pro_v7.6.0.py)

1. **バージョン情報の更新**
   - `VERSION` を `"7.6.0"` に更新。

2. **ヘッダーレイアウトの改善 (`_build_header`)**
   - `ft.Row` に `wrap=True` と `run_spacing=8` を追加し、ウィンドウ幅が狭い場合でも要素が重ならないように調整。

3. **フッターレイアウトの改善 (`_build_bottom_bar`)**
   - 参照ボタンを `ElevatedButton` に統一（コード内コメントに合わせて調整）。
   - アクションボタン群の `expand` コンテナを廃止し、`wrap=True` による自動折り返しを実装。
   - 進捗バーエリアとボタンエリアの間に `wrap=True` を設定し、幅不足時に縦に並ぶように調整。

4. **撮影者選択モーダルの改善 (`show_identity_modal`)**
   - 固定幅（280px / 600px）を撤廃。
   - `ft.ResponsiveRow` を導入し、画面サイズに応じて1列（縦並び）と2列を切り替えるレスポンシブ対応。

5. **スプリッターの動的制限 (`on_splitter_drag`)**
   - ウィンドウ幅に基づき、左ペイン（シーン選択）の最大幅を動的に算出。
   - 右ペイン（サムネイル表示）が最低 300px 確保されるように制限を追加。
