# UMIS Pro v7.2.25 撮影者・カードID選択UI統合刷新計画

## 概要
ヘッダーのカメラマンとカードIDのプルダウンを廃止し、専用の統合選択モーダルで直感的に選択・管理できる仕組みを導入します。これにより、多忙な撮影現場での操作ミスを減らし、視認性を向上させます。

## 変更内容

### 1. 状態管理の更新
- `UMISApp` クラスに `self.current_photographer` と `self.current_card_id` を明示的に保持します（初期値はプロジェクト設定から取得）。
- カードIDは、これまでドライブ名から抽出していましたが、今後はプロジェクト設定の `card_ids` リストから独立して選択する方式に一本化します。

### 2. ヘッダーUIの刷新
- `dd_photographer` と `dd_drive`（の一部機能）を担うプルダウンを削除します。
- 代わりに `self.btn_select_identity` を配置します。
    - ラベル例: `田中 / 01`
    - 未選択時: `撮影者/カードを選択`

### 3. 統合選択モーダルの実装 (`show_identity_modal`)
- **カメラマン選択エリア**:
    - 登録済みの名前をグリッド状のボタンで表示。選択中の名前をハイライト。
    - 右端に管理用アイコンを置き、名前の追加・削除ダイアログを呼び出し可能に。
- **カードID選択エリア**:
    - 「01」「02」などのIDをボタンで表示。選択中をハイライト。
    - 管理用アイコンからIDリストの編集を可能に。
- **オートクローズロジック**:
    - 両方の項目が「選択済み」の状態になった瞬間、モーダルを自動的に閉じ、メインUIを更新します。

### 4. リネームロジックの調整
- `generate_filename` および `CopyWorker` において、統合モーダルで選択された `self.current_photographer` と `self.current_card_id` を優先的に使用するように修正します。

### 5. 保存と維持の保証
- 会場（`dd_venue`）の選択状態の保存・復元が正しく行われていることを再確認し、必要に応じて修正します。

## 検証計画
- アプリケーションが起動することを確認。
- ヘッダーの新しいボタンをクリックして統合モーダルが表示されることを確認。
- カメラマンとカードIDを選択するとモーダルが自動で閉じ、ボタンのラベルが更新されることを確認。
- リネームプレビューに選択した名前とIDが即座に反映されることを確認。
- 設定がプロジェクトごとに正しく保存・読み込みされることを確認。
