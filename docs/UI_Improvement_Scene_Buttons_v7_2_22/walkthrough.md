# 修正内容の確認 (v7.2.22)

## 実施内容
シーン選択パネル内のボタンにおいて、パネル幅が狭くなった際に件数バッジが隠れてしまう問題を解決し、UIの視認性と堅牢性を向上させました。

### 1. シーンボタン構造の刷新 (Stackレイアウト)
- ボタン（ft.Container）のコンテンツを `ft.Column` から `ft.Stack` に変更しました。
- 件数バッジ（○件）を `right=0, top=0` の指定でボタンの右上に絶対配置しました。これにより、ボタン内のテキスト量やボタンの高さに関わらず、バッジが常に正しい位置に表示されます。

### 2. テキスト表示の最適化
- シーン名の表示に `max_lines=1` および `overflow=ft.TextOverflow.ELLIPSIS` を適用しました。
- シーン名が長い場合にボタンを押し広げることなく、末尾を「...」で省略することで、グリッドレイアウトの一貫性を維持します。

### 3. スタイル調整
- バッジのフォントサイズを `9` に微調整し、パディングと角丸（border_radius=8）を最適化することで、モダンでプレミアムな外観に仕上げました。

## 検証結果
- アプリケーションが正常に起動することを確認。
- シーンにファイルを割り当てた際、件数バッジがボタンの右上に重なる形で表示されることを確認。
- スプリットバーでパネル幅を最小にしても、バッジがボタン下部に押し出されることなく、視認性が維持されていることを確認。
