# UMIS Pro v7.2.12 機能改善・UI最適化実装計画

## 概要
大量のファイルを効率的に処理するための「確実な一括選択」機能の実装と、スプリットバー操作に連動した「レスポンシブなシーン表示」への最適化を行います。

1. **一括選択ロジックの再構築**: 「起点選択 → 終点選択」という明示的なステートマシンによる範囲選択を実装し、操作の確実性を向上させます。
2. **レスポンシブ・シーングリッド**: 右側パネルの幅に応じて、シーンの表示列数を自動で切り替える（1列/2列）機能を実装します。

## 変更内容

### UMIS App 本体

#### [MODIFY] [rinkan_umis_pro_v7.2.12.py](file:///Users/kiroku_keizo/開発/UMIS%20(Universal%20Media%20Ingest%20System)/安定版/rinkan_umis_pro_v7.2.12.py) [NEW]

- **VERSION更新**: `VERSION = "7.2.12"` に更新。
- **初期化 (`__init__`)**:
    - `self.range_selection_start_idx = None` を追加。
- **スプリッター操作 (`on_splitter_drag`)**:
    - `self.scene_panel.width` を監視し、250px 未満なら `self.grid_scenes.runs_count = 1`、それ以上なら `2`（または動的計算）に設定。
- **範囲選択トグル (`toggle_range_select_mode`)**:
    - モードON時に「終点を選択してください」といったツールチップの動的変更を実装。
- **ファイルクリック (`on_file_click`)**:
    - 範囲選択モードON時：起点から現在地までを一括選択し、モードを解除。
    - 範囲選択モードOFF時：通常選択を行い、クリックしたインデックスを「起点候補」として保存。

## 検証計画
- 範囲選択ボタンを押し、一つ目のファイルをクリック（起点）、二つ目のファイルをクリック（終点）してその間が全て選択されることを確認。
- スプリッターを左に動かしてシーンパネルを狭くした際、シーンのボタンが1列表示に切り替わることを確認。
- 逆に広くした際に2列（またはそれ以上）に戻ることを確認。
