# 修正内容の確認 (v7.3.0)

## 実施内容
ご要望の通り、プログラムを `v7.3.0` にアップデートし、初回起動時の動的ウォークスルー（操作ガイド）の実装と、設定モーダル内のUX強化（内部ルーティング化）を実施しました。

### 1. 動的ウォークスルー（操作ガイド）の実装
- **UIスタック構造の導入**: アプリケーション全体のルート構造を `ft.Stack` に変更し、UI全体を暗くする半透明のオーバーレイ（`tutorial_overlay`）を重ねられるように設計を変更しました。
- **初回起動判定**: `ConfigManager` に `is_first_run` フラグを持たせ、初期状態（True）の場合のみ、起動直後に自動的に `start_walkthrough()` がトリガーされる仕組みを構築しました。
- **ステップ・バイ・ステップの説明**:
  ウォークスルー中は、以下の4つのステップに沿って操作パネル付近にバルーンが表示されます。
  1. 「撮影者 / カードIDの設定」
  2. 「取り込むファイルの選択（範囲選択）」
  3. 「シーンの割り当て」
  4. 「コピー開始と初期化」

### 2. 設定モーダルのUX強化（アクション時に閉じない）
- **内部コンテンツの切り替え機構**:
  設定画面を開く処理 (`open_settings_modal`) の中で `ft.AnimatedSwitcher` を用い、モーダルの `content` を動的に入れ替えるように設計を大幅に変更しました。
- **各リストエディタの統合**:
  これまで別個の `ft.AlertDialog` を生成していた「非表示ドライブの解除」や「カメラマン管理」などの機能（`_open_list_editor`）について、メイン設定ダイアログ内でのコンテンツの書き換えと、「← 戻る」ボタンによるメイン画面への復帰機能（`_back_to_settings_main`）に置き換えました。これにより、設定操作時にモーダルが閉じてしまう問題が完全に解消されました。
- **ガイド再表示機能**:
  設定モーダルの「プロジェクト管理」内に「操作ガイドを開始」ボタンを追加しました。

### 3. 既存UIの継承
- v7.2.26 / v7.2.27 で整備された識別情報選択モーダルの「タイル型レイアウト」や、カードID等の選択ロジック、レスポンシブなシーンパネル表示などは全てそのまま引き継いでいます。

## 検証結果
- 新規プロジェクト（`is_first_run` が `True` の状態）において、起動時に正常に画面が暗転し、ウォークスルーが開始されることを確認。
- 設定モーダルから「カメラマン管理」などを開いた際、新しいダイアログが被さるのではなく、同一のウィンドウ内でシームレスに表示が切り替わり、戻るボタンで一覧に復帰できることを確認。
- スタック表示などによりエラーが発生せず、安定して起動することをテストしました。
