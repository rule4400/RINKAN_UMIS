# UMIS Pro v7.3.0 動的ウォークスルー実装および設定モーダルUX改善計画

## 概要
UI要素を強調しながら操作を説明する「オーバーレイ型ウォークスルー」を新機能として実装し、初めてのユーザーでも直感的に基本操作を理解できるようにします。また、設定画面内でリスト編集などの操作を行った際にモーダルが閉じないよう、画面切り替え方式を高度化します。

## 実装内容

### 1. ページ構造の ft.Stack 化
`build_ui` の最終的なページ出力を変更し、UI全体を `ft.Stack` でラップします。
- **メイン層 (`main_container`)**: 従来のヘッダー、メインコンテンツ、フッターを含むコンテナ。
- **チュートリアル層 (`tutorial_overlay`)**: メイン層の上に被さる半透明のコンテナ（初期状態は `visible=False`）。背景色を `#99000000` に設定し、UI全体を暗くします。

### 2. 動的ウォークスルーの実装
- **ロジック構築**: `start_walkthrough()`, `next_step()`, `end_walkthrough()` メソッドを新設します。
- **ステップ構成**:
    1. **撮影者/カード設定**: ヘッダーの統合ボタン付近にバルーンを表示。
    2. **範囲選択**: 中央のファイルリスト（サムネイル部分）付近にバルーンを表示。
    3. **シーン割当**: 右側のシーンパネル付近にバルーンを表示。
    4. **安全初期化**: 画面下部の「カード初期化」ボタン付近にバルーンを表示。
- **UIの強調**: 各ステップにおいて、対象となるUIコンポーネント（のコピー、あるいは対象要素自体の `z_index` を上げる等の工夫）をオーバーレイの上に重ねる、または対象位置に透明な枠とバルーンを配置することで、該当部分を強調します。
- **起動判定**: `ConfigManager` に `is_first_run`（初期値 `True`）を設け、初回起動時に自動で `start_walkthrough()` を呼び出し、完了時に `False` に保存します。

### 3. 設定モーダルのUX強化（内部ルーティング化）
- **同一ダイアログ内での画面遷移**:
    「カメラマン管理」「会場リスト」「除外フォルダ」などのリスト編集アクション（`_open_list_editor`）を開く際、新しい `AlertDialog` を開くのではなく、既存の `open_settings_modal` のコンテンツ領域 (`ft.Stack` または `ft.AnimatedSwitcher` 的な実装) 内で表示を切り替えます。
- これにより、編集が終わったあとに「戻る」ボタンで元の設定項目一覧にスムーズに復帰でき、モーダル自体が閉じてしまうストレスを解消します。
- **ガイド再表示機能**:
    設定モーダルの「プロジェクト管理」セクション内に「操作ガイド（ウォークスルー）を開始」ボタンを追加します。クリック時にモーダルを閉じ、`start_walkthrough()` をトリガーします。

### 4. 既存機能の維持
- `v7.2.26` 以降の左右2カラムタイル型レイアウト（識別情報）や、バッジ表示機能、レスポンシブなカラム数切り替えなどは完全な形で維持します。

## 検証計画
- `v7.3.0` を初回起動した際に、自動的に画面が暗転し、オーバーレイ型のウォークスルーが開始されることを確認。
- ウォークスルーの「次へ」をクリックしていくと、ステップごとにバルーンの位置や説明文が変化し、最後に通常画面に戻ることを確認。
- 設定モーダルから各種リストエディタを開いた際、モーダルが閉じずに内部のコンテンツのみが切り替わることを確認。
- 設定モーダルから「操作ガイドを開始」をクリックし、再びウォークスルーが実行されることを確認。
