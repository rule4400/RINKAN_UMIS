# 修正内容の確認 (v7.3.9 UI安定化・配置最適化・スクロール完全実装)

## 実施内容
画面解像度の違いによる表示崩れや、設定画面内でのクリック不備を徹底的に解消した `v7.3.9` をリリースしました。

### 1. 配置の最適化と「上揃え」固定
- **一貫したレイアウト**: カスタム設定画面において、ウィンドウの高さがある場合でも項目が中央に浮いてしまわないよう、常に「上揃え（START）」で配置されるように修正しました。
- **ヘッダーの折り返し**: 幅の狭い画面でも「設定」や「履歴」ボタンが外に突き抜けないよう、自動で2段に折り返される設定（wrap=True）を適用しました。

### 2. 設定画面の「i」ボタン（ヘルプ）の完全復活
- **クリック反応性の向上**: ヘルプアイコンを格納するコンテナの設定を見直し、クリックイベントが正確に発火するように修正しました。
- **干渉の排除**: 背面のオーバーレイ等がクリックを吸い取ってしまう問題を解決し、設定項目ごとの詳細なヒントを確実に確認できるようになりました。

### 3. あらゆる解像度に対応するスクロール設計
- **メインスクロールの導入**: アプリ全体を包むルートコンテナにスクロール機能を実装しました。これにより、実行ボタンが画面下部に隠れて押せなくなるという致命的な問題を根本的に解決しました。
- **設定パネルの内スクロール**: 設定モーダルの高さを画面の90%に動的に制限しました。これにより、どんなに項目が増えても、あるいは画面が低くても、必ずパネル内部でスクロールが発生し、最下部のボタンまでアクセス可能です。

### 4. 機能とデザインの継承
- 白色（WHITE70）の細い枠線によるスポットライト効果と、主要5ステップに絞った操作ガイドは引き続き搭載されています。
- 右クリック（2本指）範囲選択やシーンバッジ等の基幹機能もすべて正常に動作します。

## 検証結果
- 設定画面内での各「i」ボタンの正常動作を確認しました。
- ウィンドウを極端に縮小した際にも、スクロールバーを介して全ボタン・ログにアクセスできることを確認済みです。
