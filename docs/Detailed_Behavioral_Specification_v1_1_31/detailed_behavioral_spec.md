# RINKAN UMIS v1.1.31 詳細挙動仕様書

## 1. アプリケーションモード (App Mode)

### 1.1 取り込みモード (Ingest Mode)
- **目的**: 外部メディア（SDカード等）からPCへファイルを整理しながらコピーする。
- **挙動**: 
    - サイドバーには「メディア（接続ドライブ）」が表示される。
    - 中央エリアにはソースメディア内のファイルがグリッド/リスト形式で表示される。
    - 右パネル（インスペクタ）には、選択したファイルのメタデータとシーン割当ボタンが表示される。

### 1.2 選別モード (Select Mode)
- **目的**: 取り込み済みのライブラリから、編集・納品用に使用するファイルを抽出する。
- **挙動**:
    - サイドバーには「ライブラリ（シーン別フォルダ）」が表示される。
    - 中央エリアには保存済みファイルが表示される。
    - ファイルごとに「選別フラグ（星アイコン）」を付与でき、フラグが立ったファイルのみを一括出力できる。

## 2. メディア検出とスキャンの挙動

### 2.1 自動スキャン
- 1秒間隔の `drive_monitor` が新しいドライブを検知。
- ドライブ選択時、`_scan_worker` がバックグラウンドで開始。
- **条件分岐**:
    - `excluded_folders`（設定画面で定義）に含まれるフォルダは完全にスキップ。
    - ファイル数が5000件、または総容量が128GBを超える場合、スキャン実行前に確認ダイアログを表示。
    - スキャン中もUI操作は可能（非同期処理）。

### 2.2 カテゴリ分類規則
拡張子に基づいて以下の4カテゴリに自動分類される。
- **Photo**: `.jpg`, `.jpeg`, `.png`, `.heic` 等
- **Movie**: `.mp4`, `.mov`, `.mxf`, `.mts` 等
- **Raw**: `.arw`, `.cr2`, `.cr3`, `.dng`, `.nef` 等
- **Audio**: `.wav`, `.mp3`, `.m4a` 等

## 3. ファイル選択とシーン割当の挙動

### 3.1 選択操作
- **単一選択**: クリックで選択。
- **範囲選択 (Shift+Click)**: 最後に選択した位置から現在位置までを一括選択。
- **複数選択 (Cmd/Ctrl+Click)**: 選択状態を反転。
- **全選択 (Cmd/Ctrl+A)**: 表示中の全ファイルを選択。

### 3.2 シーン割当ロジック
- 選択中のファイルに対し、シーンボタンをクリックすると `assigned_scene` プロパティが更新される。
- **重複割当**: すでに割り当て済みのファイルを別のシーンへ割り当て直すと、上書きされる。
- **解除**: 割当済みのファイルを右クリックメニューまたは「割当解除」ボタンでリセット可能。

## 4. リネームと整理の挙動 (Renaming Logic)

### 4.1 ファイル名生成アルゴリズム
`generate_filename` メソッドにより、以下の優先順位とスイッチの組み合わせで生成される。

| 要素 | スイッチ名 | 取得内容 |
| :--- | :--- | :--- |
| 日付 | `sw_rename_date` | EXIFまたはファイル更新日時。形式は `dd_date_format` に従う。 |
| 会場 | `sw_rename_venue` | `dd_venue` で選択された値。 |
| シーン | `sw_rename_scene` | 割当済みシーンの名称。 `sw_scene_numbering` がONならDayNoを付与。 |
| カメラマン | `sw_rename_pg` | `dd_photographer` で選択された名称。 |
| カードID | `sw_rename_id` | `dd_card_id` で選択されたID。 |
| 連番/元名 | `sw_rename_seq` | ON: 0001からの連番、OFF: 元のファイル名。 |

### 4.2 フォルダ階層構造
`[保存先ルート] / [シーン名] / [カテゴリ名] / [カメラマン名] / [リネーム済みファイル]`
- シーン名に番号を付与する設定（`scene_numbering`）が有効な場合、フォルダ名も `101_会場到着` のようになる。

## 5. コピー処理 (CopyWorker) の内部挙動

### 5.1 フェーズ1: 一時転送 (Phase 1: Transfer)
- `TEMP_DIR_FIXED` (ユーザーのホームディレクトリ直下) 内に、セッションごとの一時フォルダを作成。
- ソースメディアから一時フォルダへそのままの名称でコピー。
- **中断時**: `is_cancelled` がTrueになると即座にループを抜け、中途半端なファイルは削除を試みる。

### 5.2 フェーズ2: 整理と検証 (Phase 2: Organize & Validate)
- 一時フォルダから最終保存先へ、リネームを適用しながら移動。
- **競合回避**: 同一フォルダ内に同名ファイルが存在する場合、 `_v1`, `_v2` ... を付与。
- **検証**: 移動後のファイルサイズがソースと一致するかチェック。不一致ならエラーを記録。

## 6. カード初期化 (Format) の安全挙動

### 6.1 多段階警告
1. **未割当警告**: シーンに割り当てられていない（取り込まれない）ファイルが残っている場合、警告を表示。
2. **容量警告**: 128GB以上のドライブの場合、SSDやHDDを誤ってフォーマットしないよう「大容量ドライブ警告」を表示。
3. **3段階確認**: 「初期化確認 (1/3)」→「再確認 (2/3)」→「最終確認 (3/3)」を経て実行。

### 6.2 実行
- macOS: `diskutil eraseDisk ExFAT [Label] MBRFormat [Device]`
- Windows: `format [DriveLetter]: /FS:exFAT /Q /V:[Label]`

## 7. ショートカットとプレビュー操作

### 7.1 ファイル一覧画面
- **Space**: クイックプレビュー開始。
- **Esc**: クイックプレビュー終了。
- **Enter / S**: 選別モード時にフラグ（星）をトグル。
- **Arrows (左右上下)**: 選択ファイルの移動。

### 7.2 プレビュー中
- **J / L**: 5秒戻る / 5秒進む。
- **K / Space**: 再生 / 一時停止。
- **0〜9**: 再生位置を10%刻みでジャンプ。
- **M**: 消音（ミュート）。

---

## 8. キャッシュ管理の挙動

- **サムネイル**: 200pxサイズの画像を `thumb_cache/` に保存。
- **プレビュー**: 1080pサイズの画像を `previews/` に保存（画像のみ）。
- **Durationキャッシュ**: `duration_cache.json` に動画パスと秒数を記録。2秒間のデバウンス処理（`_schedule_duration_cache_save`）を挟んでディスクに保存することで、連続スキャン時の負荷を軽減。
