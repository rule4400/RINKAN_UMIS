# UMIS Pro v7.2.21 操作体系の確定とバグ修正計画

## 概要
Flet環境におけるShiftキー検知の不安定さを完全に排除し、マウス操作のみで確実かつ爆速に範囲選択ができる操作体系に移行します。また、設定画面が表示されないバグ（AttributeError）の修正と、ドライブ変更時のUI状態の徹底した初期化を行います。

## 変更内容

### 1. Shiftキー依存の完全排除
- **UI/変数削除**: `self.lbl_key_debug`（デバッグラベル）、`self.shift_pressed`（フラグ）を削除。
- **イベント削除**: `build_ui` 内の `on_keyboard` イベントハンドラを完全に削除。

### 2. 右クリック範囲選択の正式採用
- **左クリック (`on_file_click`)**:
    - 単体選択のトグルのみ行い、クリックしたインデックスを `self.range_selection_start_idx`（範囲選択の起点）に保存。
- **右クリック (`on_file_right_click`)**:
    - 新規メソッドとして実装。起点から右クリック地点までの範囲をすべて選択状態（`selected = True`）にする。
- **ビューの更新**: `_refresh_grid_view` および `_refresh_list_view` の各アイテムに `on_secondary_tap`（右クリック）を登録。

### 3. 設定モーダルの修正 (AttributeError 対策)
- **変数定義**: `self.rename_labels` が未定義だったため、`__init__` にて以下の通り定義。
    - `{"date": "撮影日", "location": "会場名", "scene": "シーン名", "photographer": "カメラマン", "card_id": "カードID"}`
- **表示の安定化**: `open_settings_modal` にて `self.page.open(dlg)` 直前に `self.page.update()` を追加し、表示を保証。

### 4. ドライブ変更時のUI初期化
- `_start_scan` メソッドにて、`self.selected_scene_info = None` だけでなく、シーン選択のラジオボタンやプレビュー表示の初期化も徹底。

## 検証計画
- アプリケーションが起動することを確認。
- キーボードを押してもデバッグ表示が出ない（削除されている）ことを確認。
- サムネイルを左クリック（起点）し、別のサムネイルを右クリックして、その間が一括選択されることを確認。
- 設定アイコンをクリックした際に、エラーなく設定モーダルが表示されることを確認。
- ドライブを変更した際に、前のソースの選択状態やシーンの紐付けが消えていることを確認。
